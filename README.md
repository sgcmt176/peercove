# PeerCove

[![CI](https://github.com/sgcmt176/peercove/actions/workflows/ci.yml/badge.svg)](https://github.com/sgcmt176/peercove/actions/workflows/ci.yml)

**事業者サーバーを持たない、招待制のプライベート VPN。**

メンバーの 1 人(ホスト)の PC がコーディネーター兼リレーを担い、トンネルには
WireGuard プロトコルを使用します。アカウント登録・クラウドサービス・月額課金は
不要で、通信と設定はメンバーの端末の中だけで完結します。

## 主な機能

- **かんたんな招待制ネットワーク** — ホストが招待トークンを発行(QR コード /
  `peercove://` 参加リンク対応)し、相手は貼り付けるだけで参加。招待には
  期限・一回利用・参加承認を設定できます
- **デスクトップアプリ + 常駐デーモン** — トレイ常駐・OS 通知・ライト/ダーク
  テーマ。アプリを閉じても接続は維持され、OS 起動時に自動復帰します。
  複数ネットワークの同時稼働にも対応
- **メンバー間の直接通信** — 可能な環境では NAT を越えて 1 ホップで直接通信し、
  できない場合はホスト経由へ自動フォールバック。経路はバッジで確認できます
- **トンネル内 DNS** — `<名前>.<ネットワーク名>.peercove.internal` でメンバーや
  LAN 機器へアクセス。サービス URL の共有・死活監視(ヘルスチェック)付き
- **チャットとファイル送信** — 1:1 / 全体 / グループ。画像・動画・テキストの
  プレビュー、ドラッグ&ドロップ対応。履歴は各端末のローカルにのみ保存
- **サブネットルーター** — メンバーの背後の LAN 機器(NAS など、PeerCove を
  入れられない機器)へ全員がトンネル越しに到達
- **通信制御(ACL)** — メンバー・グループ・サブネット間の通信を、方向・
  プロトコル・ポート単位のルールで許可 / 拒否
- **運用機能** — 接続診断センター、通信品質履歴(RTT・損失・経路)、
  デバイス鍵の自動ローテーション、暗号化バックアップと端末移行、更新通知

各機能の使い方は [docs/user-guide.md](docs/user-guide.md) を参照してください。

## 対応環境

| OS | 役割 | 備考 |
|---|---|---|
| Windows 10 / 11 | ホスト・メンバー | TUN ドライバ(wintun)はインストーラに同梱 |
| Ubuntu 22.04+ | ホスト・メンバー | カーネル標準の WireGuard モジュールを使用 |

macOS・モバイル・IPv6 は現時点では未対応です。

## インストール

[GitHub Releases](https://github.com/sgcmt176/peercove/releases) から入手します。

- **Windows**: `PeerCove_x.y.z_x64_ja-JP.msi` を実行。デーモンのサービス登録・
  ファイアウォール許可・TUN ドライバの配置まで自動で行われます
- **Ubuntu**: `sudo apt install ./PeerCove_x.y.z_amd64.deb`。systemd サービスが
  登録・起動されます
- **ポータブル版(上級者向け)**: ZIP / tar.gz。セットアップは手動です
  (同梱の README-portable を参照)

アップデート時は**全端末のアプリとデーモンを対で更新**してください(新機能の
設定を含む構成ファイルは旧バージョンでは読めないことがあります。ホストを先に
更新するのが安全です)。

ソースからビルドする場合は [docs/development.md](docs/development.md) を参照。

## はじめかた

1. **ホスト側**: PeerCove を起動 → 「新しくホストする」→ ネットワーク名を付けて
   作成・接続(鍵・トンネル用サブネットは自動生成。UPnP によるルーターの
   ポート開放も自動で試みます)
2. **ホスト側**: 「メンバーを招待」→ 名前を付けて発行 → トークン(または
   参加リンク・QR)を相手へ渡す
3. **メンバー側**: PeerCove を起動 → 「招待トークンで参加する」に貼り付けて参加
4. 以降はメンバー一覧・チャット・ファイル送信・DNS 名でのアクセスが使えます

CLI だけでも一通りの操作ができます([docs/user-guide.md](docs/user-guide.md) の
「CLI の使い方」)。

> **招待トークンは秘密情報です。** 参加する本人以外へ渡さず、使用後は破棄して
> ください(既定で 7 日の期限付き・一回使うと無効になります)。

## セキュリティとプライバシー

- トンネルは WireGuard プロトコル(Curve25519 / ChaCha20-Poly1305)で暗号化
  されます。直接通信が成立している相手とはエンドツーエンド暗号化、ホスト経由の
  通信はホストで復号・再暗号化されます(ホストはネットワークの管理者です)
- 参加後、デバイスの秘密鍵は端末上で自動的に再生成(ローテーション)され、
  以後**秘密鍵が端末の外へ出ることはありません**
- 直接通信を使う場合、あなたの外部 IP アドレスが同じネットワークのメンバーに
  共有されます(設定でオフにできます)
- チャット履歴・受信ファイル・通信品質履歴はすべて各端末のローカルにのみ
  保存されます。外部の計測・診断サーバーは使用しません
- 秘密鍵・事前共有鍵・招待トークンはログに出力されません

## トラブルシューティング(よくあるもの)

- **メンバーと繋がらない(handshake が成立しない)** — ホスト側の
  ファイアウォールで UDP 51820(トンネル)と TCP 51821(メンバー一覧の配布)の
  受信が許可されているか確認してください。インストーラ導入なら自動で許可されます
- **同一 LAN 内で繋がらない** — 招待時のエンドポイントに外部 IP ではなく
  ホストの LAN IP を使ってください(ルーターのヘアピン NAT 非対応のため)
- **Tailscale が入っている端末で ping が届かない** — 仮想 IP のレンジが
  衝突している可能性があります。詳細は
  [docs/verification.md](docs/verification.md) の G-2 の確認ポイントを参照
- **アプリが「デーモンの更新が必要です」と表示する** — アプリとデーモンの
  バージョンがずれています。[docs/user-guide.md](docs/user-guide.md) の
  「トラブルシューティング: デーモンが古い」を参照

原因が分からないときは、ネットワーク詳細の「診断」(接続診断センター)が
設定・トンネル・DNS・ACL を確認して原因の候補を表示します。
