# 機能計画書の命名規約

追加開発の計画書は、このディレクトリへ次の名前で保存する。

```text
YYYY-MM-DD-<theme>.md
```

- 日付は計画を最初に作成した日とする。
- `theme` は対象機能を表す短い英小文字の kebab-case とする。
- 計画の対象が後から増えてもファイル名は変更せず、文書内の対象欄を更新する。
- 同じ日に別テーマの計画を作る場合は `theme` で区別する。
- 既存計画の改訂版を別文書にする必要がある場合だけ、末尾へ `-v2` などを付ける。

例:

```text
2026-07-14-vpn-operations.md
2026-08-03-device-management.md
2026-08-03-performance.md
```
